# Templates

Invoice PDFs are generated by [Typst](https://typst.app/), a modern typesetting system. The default template produces a clean, professional invoice. You can customize it or create entirely different templates per entity or project.

## Template resolution order

1. **Project template** — `template` field in `projects/<slug>.yaml`
2. **Entity template** — `template` field in the entity config
3. **Default** — `templates/invoice.typ`

The first one found wins. All template filenames are relative to the `templates/` directory.

## Setting a template

### During setup

All three interactive commands ask for a template:

```bash
consult init                    # for the first entity
consult entity add              # for additional entities
consult entity edit <slug>      # to change later
consult project new             # per-project override
```

The prompt:
```
Invoice template (filename in templates/, or enter for default):
```

Just press enter to use the stock template. Or type a filename:
```
Invoice template: fancy-client.typ
```

### Manually

Edit `config.yaml` for entity-level:
```yaml
entities:
  clairlea:
    template: clairlea-branded.typ
```

Edit `projects/<slug>.yaml` for project-level:
```yaml
template: acme-custom.typ
```

---

## Creating a custom template

Start by copying the default:

```bash
cp templates/invoice.typ templates/my-custom.typ
```

Then edit `templates/my-custom.typ` to your needs.

## Template data

Your template receives a YAML file containing all invoice data. Access it with:

```typst
#let data = yaml(sys.inputs.data)
```

### Available fields

```yaml
invoice_number: "2026-001"
invoice_date: "2026-07-15"

entity:
  name: "Clairlea Consulting"
  business_number: "827-443-656-RT0001"   # or null
  email: "morgen@example.com"
  phone: "555-1234"                        # or null
  payment_terms: "Net 30 days"

client: "Johnny Smith"
project: "Demo-Comp dot CA"
currency: "CAD"
rate_type: "daily"             # or "hourly"
hours_per_day: 7.5             # or null for hourly

line_items:                    # array of entries
  - date: "2026-06-23"
    hours: 4.0
    rate: 31.87
    focus: "Frontend migration"
    tax: 13                    # number or "exempt"
    amount: 127.48

taxable_subtotal: 557.73
exempt_subtotal: 0.0

tax_breakdown:                 # one entry per distinct tax rate
  - percent: 13
    subtotal: 557.73
    tax: 72.50

total_tax: 72.50
grand_total: 630.23
notes: "Optional invoice notes"   # or null
```

### Accessing data in Typst

```typst
#let data = yaml(sys.inputs.data)
#let entity = data.entity
#let line_items = data.line_items

// Entity info
#entity.name
#entity.business_number

// Iterate line items
#for item in line_items {
  // item.date, item.hours, item.rate, item.amount, item.focus, item.tax
}

// Totals
#data.grand_total
#data.currency
```

## Default template walkthrough

The stock `templates/invoice.typ` has these sections:

### 1. Header

Two-column grid: entity info on the left, "INVOICE" title with number and date on the right.

### 2. Bill To / Terms

Two-column grid: client name and project on the left, payment terms on the right.

### 3. Line items table

Dynamic table with columns: Date, Hours, Rate, Amount, Description. Tax-exempt items are marked with `†`.

### 4. Totals

Right-aligned summary table: subtotal, tax breakdown by rate, grand total. If exempt entries exist, shows taxable/exempt subtotal breakdown.

### 5. Notes

Optional section, only rendered if `data.notes` is not null.

### 6. Footer

Entity name and business number.

## Typst tips

### Escaping dollar signs

In Typst, `$` starts math mode. Use `\$` for literal dollar signs:
```typst
[\$#str(data.grand_total)]    // renders: $630.23
```

Or build strings in code:
```typst
#let amount_str = "\$" + str(calc.round(item.amount, digits: 2))
```

### Conditional sections

```typst
#if data.notes != none [
  Notes: #data.notes
]

#if entity.business_number != none [
  Business #: #entity.business_number
]
```

### Number formatting

```typst
#str(calc.round(item.hours, digits: 2))    // "4.00" — note: may show "4.0"
```

For guaranteed 2 decimal places, you may need a helper:
```typst
#let fmt2(n) = {
  let s = str(calc.round(n, digits: 2))
  if "." not in s { s + ".00" }
  else if s.split(".").last().len() == 1 { s + "0" }
  else { s }
}
```

### Page settings

```typst
#set page(paper: "us-letter", margin: (top: 18mm, bottom: 18mm, left: 20mm, right: 20mm))
```

Change to `"a4"` for international use.

### Fonts

```typst
#set text(size: 10pt, font: "Inter")
```

The font must be installed on your system. Omit the `font` parameter to use Typst's default.

## Example: minimal template

```typst
#let data = yaml(sys.inputs.data)

#set page(paper: "us-letter", margin: 2cm)
#set text(size: 11pt)

= INVOICE #data.invoice_number

*#data.entity.name* \
Date: #data.invoice_date

---

*Bill to:* #data.client \
*Project:* #data.project

#table(
  columns: (auto, auto, auto, 1fr),
  [*Date*], [*Hours*], [*Amount*], [*Description*],
  ..data.line_items.map(item => (
    [#item.date],
    [#str(item.hours)],
    [\$#str(calc.round(item.amount, digits: 2))],
    [#item.focus],
  )).flatten()
)

#align(right)[
  *Total: \$#str(calc.round(data.grand_total, digits: 2)) #data.currency*
]
```

## Debugging templates

If PDF generation fails, the intermediate data file is preserved at `build/<invoice-number>.yaml`. You can:

1. Inspect the data: `cat build/2026-001.yaml`
2. Test the template directly: `typst compile templates/invoice.typ test.pdf --root . --input "data=/build/2026-001.yaml"`
3. Fix the template and re-run the typst command until it works
